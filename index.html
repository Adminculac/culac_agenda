ติดตามเรื่องของฉัน

ตรวจสอบสถานะและมติที่ประชุมของเรื่องที่งานของท่านเสนอเข้ามา

วาระการประชุม

ดูรายการเรื่องที่เสนอเข้าที่ประชุม อัปเดตล่าสุดจากผู้รวบรวมวาระ

ส่งเรื่องเข้าที่ประชุม

กรอกรายละเอียดเรื่องที่ต้องการเสนอเข้าที่ประชุมคณะกรรมการบริหาร พร้อมแนบไฟล์เอกสารประกอบ

วาระประจำ = เรื่องที่ต้องเสนอทุกครั้ง เพียงเลือกหัวข้อแล้วแนบไฟล์
ขึ้นต้นด้วยคำกริยาให้ชัดเจน เช่น “ขออนุมัติ…”, “รายงาน…”
📎
คลิกเพื่อเลือกไฟล์ หรือลากไฟล์มาวางที่นี่
รองรับ PDF, Word, รูปภาพ ฯลฯ · ต่อไฟล์ไม่เกิน 10 MB

ส่งเรื่องเรียบร้อยแล้ว

เรื่องของท่านถูกส่งถึงผู้รวบรวมวาระเรียบร้อยแล้ว

การประชุม:

เรื่องที่เสนอเข้าที่ประชุม

ตรวจสอบ จัดสถานะ บันทึกมติ และแก้ไขเรื่องที่แต่ละงานส่งเข้ามา

ประเภทเรื่อง / งาน / หมวดงานผู้เสนอ ไฟล์สถานะมติที่ประชุมจัดการ

รายงาน

รายงานสำหรับผู้บริหารและการติดตามงาน — เลือกรอบประชุมด้านบน หรือดูภาพรวมข้ามรอบ

จัดระเบียบวาระและพิมพ์เอกสาร

จัดลำดับเรื่องที่ “บรรจุแล้ว” และพิมพ์ระเบียบวาระตามรูปแบบราชการ

ปุ่ม ▲▼ ในแต่ละหมวดใช้จัดลำดับเรื่อง · เรื่องที่จะขึ้นเอกสารคือสถานะ “บรรจุแล้ว” (และ “รอบรรจุ” หากติ๊กด้านบน)

รอบการประชุม

สร้างและจัดการรอบการประชุม แต่ละงานจะเห็นเฉพาะรอบที่ “เปิดรับเรื่อง”

ครั้งที่วัน–เวลา / ช่องทางเรื่อง สถานะจัดการ

ตั้งค่าระบบ

เชื่อมต่อฐานข้อมูลกลาง (GAS) สำรองข้อมูล และตั้งค่าเอกสาร

การเชื่อมต่อฐานข้อมูลกลาง (Google Apps Script)

วางลิงก์ Web App (ลงท้าย /exec) เพื่อให้ทุกงานส่งเรื่องมารวมที่เดียว หากเว้นว่างจะทำงานแบบเก็บในเครื่องนี้เท่านั้น

🔗 ลิงก์สำหรับแจกให้ทุกงาน

ใช้ลิงก์นี้แจกให้เจ้าหน้าที่ทุกคน เปิดแล้วเชื่อมต่อฐานข้อมูลกลางได้ทันที ไม่ต้องตั้งค่าเอง (เปิดครั้งแรกครั้งเดียว ระบบจะจำไว้ให้)

รหัสผ่านผู้ดูแล

ใช้กันคนทั่วไปเข้าโหมดผู้ดูแล (ป้องกันระดับพื้นฐาน)

โลโก้

แยกกัน 2 ชุด — โลโก้ระบบแสดงบนแถบด้านบนของหน้าเว็บ ส่วนตราสัญลักษณ์แสดงบนหัวเอกสารระเบียบวาระ

สำรอง / กู้คืนข้อมูล

ดาวน์โหลดข้อมูลทั้งหมดเป็นไฟล์ JSON หรือกู้คืนจากไฟล์สำรอง

แจ้งเตือนทางอีเมลเมื่อมีเรื่องส่งเข้ามา

ระบบจะส่งอีเมลแจ้งทันทีที่มีการส่งเรื่องเข้ามา เหมาะกับเรื่องด่วนที่ต้องเวียนขอมติ (ทำงานฝั่ง Google จึงแจ้งได้แม้ไม่ได้เปิดหน้าเว็บไว้)

หลายคนคั่นด้วยเครื่องหมายจุลภาค (,)
ใส่ลิงก์หน้าเว็บที่เผยแพร่แล้ว เพื่อให้ผู้รับกดเข้าระบบได้จากอีเมลโดยตรง

ตรวจสอบระบบ

ตรวจการเชื่อมต่อฐานข้อมูลกลาง จำนวนข้อมูล และความพร้อมของคอลัมน์ที่ระบบต้องใช้

สร้าง QR Code สำหรับลิงก์ส่งเรื่อง

วางลิงก์หน้าเว็บที่เผยแพร่แล้ว (เช่น GitHub Pages) เพื่อสร้าง QR ให้ทุกงานสแกนเข้าฟอร์มส่งเรื่อง — ติดบอร์ด แนบบันทึกเวียน หรือส่งในไลน์/อีเมลได้

ควรเป็นลิงก์สาธารณะที่ทุกงานเปิดได้ ไม่ใช่ไฟล์ในเครื่อง

เข้าสู่โหมดผู้ดูแล

รหัสผ่านจะแสดงเป็นจุด เพื่อความปลอดภัยขณะนำเสนอ

เพิ่มเรื่อง

วันสุดท้ายที่ต้องได้มติเพื่อดำเนินการต่อทัน — หากใกล้ถึงกำหนดและยังไม่ประชุม ควรพิจารณาเปลี่ยนเป็นเรื่องด่วนขอเวียนมติ

บันทึกผลการประชุม

เรื่องด่วนที่เวียนขอมติ ให้บันทึกวันที่เวียนและผลไว้ในช่องนี้

สร้างการประชุม

คำนวณอัตโนมัติจากวันที่ประชุม (ปีงบประมาณ ต.ค.–ก.ย.)
เลือกจากปฏิทิน ระบบจะแปลงเป็นข้อความไทยให้อัตโนมัติ
